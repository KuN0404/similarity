Fitur Utama

Mengapa SISINDO?

Repository UUID-based

Setiap dokumen menggunakan UUID v4 sebagai identitas unik. Metadata tersimpan di MariaDB dengan struktur terorganisir per tipe file.

Manual Indexing

Indexing dilakukan manual dengan kontrol penuh. Admin dapat memulai dan membatalkan proses kapan saja dengan tombol di admin panel.

Dual Source Checking

Cek similarity terhadap dokumen lokal (repository kampus) dan internet (Google Search) dengan algoritma tokenisasi NLTK yang akurat.

Color-coded Highlighting

Setiap sumber plagiarisme di-highlight dengan warna unik pada dokumen hasil. Mudah diidentifikasi dan dipahami.

Detailed Reports

Laporan lengkap berisi summary, source breakdown, tabel kalimat terindikasi, dan kategori plagiarisme dalam format DOCX.

Role Management

Sistem role untuk Super Admin, Staff, dan Mahasiswa dengan permission berbeda dan limit upload per bulan untuk mahasiswa.

Alur Kerja

Bagaimana SISINDO Bekerja?

1

Upload Dokumen

Admin atau mahasiswa upload file PDF/DOCX ke repository dengan metadata lengkap.

2

Manual Indexing

Admin trigger indexing manual. Sistem ekstrak teks dan simpan sebagai .content.txt dengan UUID.

3

Cek Plagiarisme

Upload/paste dokumen, pilih sumber (Lokal/Internet/Both), sistem analisis per kalimat dengan tokenizer NLTK.

4

Dapatkan Laporan

Download dokumen ter-highlight + laporan detail dengan breakdown per sumber dan kategori plagiarisme.
